Diseños por Bloques

Introducción

En cualquier experimento, la variabilidad que surge de un factor puede afectar los resultados, pero cuando este factor no es de interés o no hace parte del tratamiento a evaluar se le considera como factor perturbador. Por lo general este tipo de factores son desconocidos y no controlables, a través de la aleatorización se puede controlar en gran medida los efectos de estos factores sobre la variable respuesta.

Existen casos en que estos factores son conocidos y se pueden controlar, no obstante, no son de interés para el comportamiento de la variable respuesta. Por ejemplo, si se quiere probar un tratamiento para un producto que es fabricado por cuatro máquinas, cada una de estas es operada por una persona distinta.

Operadores


El interés aquí es ver cómo el tratamiento influye sobre la respuesta de las máquinas, no obstante, el efecto que puede hacer el operador puede influir en la variabilidad del resultado. Para controlar esta variabilidad se trata al efecto del operador como un bloque, es decir, un factor adicional que no es de interés pero que afecta los resultados. Para controlar la variabilidad del bloque lo que generalmente se hace es aleatorizar el diseño incluyendo el nuevo factor, así cada operador manipula cada máquina.

Bloques


De esta manera se elimina sistemáticamente el efecto del bloque sobre las comparaciones estadísticas entre los tratamientos. Este nuevo “factor” se incluye en el ANOVA, si resulta significativa implica que el factor de bloques tiene influencia sobre la variable de respuesta, y debe ser tomado en cuenta para mejorar la calidad de ésta. Pero, si no se rechaza y se acepta que los bloques son iguales en respuesta media, entonces se tiene el argumento a favor de no controlar este factor en futuros experimentos sobre esta misma respuesta, además de que su influencia en la calidad de la respuesta no es significativa.

Modelo Estadístico

Para aplicar los bloques en un diseño factorial, se recomienda utilizar los diseños por bloques completamente aleatorizados (DBCA). Cuando se utiliza este diseño, el experimentador piensa que cada medición será el resultado del efecto del tratamiento donde se encuentre, del efecto del bloque al que pertenece y de cierto error que se espera sea aleatorio. El modelo estadístico para este diseño está dado por:

\[Y_{ij}=\mu+\tau_i+\gamma_j+\varepsilon_{ij}\\i=1,2,\dots,k \quad j=1,2,\dots,b\]

Donde \(Y_{ij}\) es la medición que corresponde al tratamiento \(i\) y al bloque \(j\), \(\mu\) es la media global poblacional, \(\tau_i\) es el efecto del tratamiento \(i\), \(\gamma_j\) es el efecto del bloque \(j\) y \(\varepsilon_{ij}\) es el error aleatorio atribuible a la medición \(Y_{ij}\), donde

\[\varepsilon_{ij}\sim NID(0,\sigma^2)\]

Para este diseño las hipótesis a probar son:

\[H_0: \mu_1=\mu_2=\dots=\mu_k=\mu\\ H_a: \mu_i \neq \mu_j \quad \text{para algun} i\neq j\]

O en términos de de los tratamientos:

\[H_0: \tau_1=\tau_2=\dots=\tau_k=0\\ H_a: \tau_i \neq 0 \quad \text{para algun}\,\, i\]

En cuanto a las hipótesis de para los bloques se plantea:

\[H_0: \gamma_1=\gamma_2=\dots=\gamma_b = 0\\ H_a: \gamma_j \neq 0 \quad \text{para algun bloque} \,\, j\]

Las hipótesis se prueban a través del Análisis de Varianza (ANOVA) con dos criterios de clasificación ya que se controlan dos fuentes de variación: los tratamientos y los bloques.

Fuente de Variación SS gl MS F valor-p
Tratamientos \(SSB=\sum_{i=1}^{k}\frac{Y^2_i.}{b}-\frac{Y^2_{..}}{N}\) \(k-1\) \(MSB=\frac{SSB}{k-1}\) \(F_0=\frac{MSB}{MSE}\) \(P(F>F_0)\)
Bloques \(SS\gamma=\sum_{j=1}^b \frac{Y^2_{.j}}{k}-\frac{Y^2..}{N}\) \(b-1\) \(MS\gamma=\frac{SS\gamma}{b-1}\) \(F_0=\frac{MS\gamma}{MSE}\) \(P(F>F_0)\)
Error \(SSE=SST-SSB-SS\gamma\) \((k-1)(b-1)\) \(MSE=\frac{SSE}{(k-1)(b-1)}\)
Total \(SST=\sum_{j=1}^b\sum_{i=i}^{k} Y^2_{ij}-\frac{Y^2_{..}}{N}\) \(N-1\)

Una vez se construye la tabla de ANOVA se verifican los supuestos del modelos a través de los residuales o errores aleatorios, similar a los diseños de un solo factor, entonces hay que comprobar:

  1. Normalidad
  2. Independencia
  3. Homocesdasticidad

Con los supuestos del modelos validados, se procede a realizar las pruebas post-hoc si se desea inferir sobre los niveles de los tratamientos del modelo. Se pueden utilizar los métodos:

  • Comparaciones de diferencia mínima significativa (LSD)
  • Método Tukey HSD
  • Método Duncan
  • Método Dunnet para fijar un nivel como un control

Ejemplo

Un equipo de mejora investiga el efecto de cuatro métodos de ensamble A, B, C y D, sobre el tiempo de ensamble en minutos. En primera instancia, la estrategia experimental es aplicar cuatro veces los cuatro métodos de ensamble en orden completamente aleatorio (las 16 pruebas en orden aleatorio). Cada método de ensamble es llevado a cabo por cuatro operadores, y se pretende controlar este factor mediante un bloque, entonces se plantea un diseño por bloque completemanete al azar (DBCA), donde cada operario aplicará cada método de ensamble, se supone que, además del método de ensamble y el bloque, no existe ningún otro factor que influya de manera significativa sobre la variable de respuesta (tiempo de ensamble). Los tiempos de ensamble obtenidos son:

Método de Ensamble
Método Operador
1 2 3 4
A 6 9 7 8
B 7 10 11 18
C 10 16 11 14
D 10 13 11 9

El modelo a evaluar es:

\[y_{ij}=\mu+\tau_i+\gamma_j+\varepsilon_{ij}\qquad \varepsilon_{ij}\sim N(0,\sigma^2)\]

Utilizando R el análisis estadístico de este diseño experimental es sencillo. Los pasos a seguir son los siguientes:

  1. Cargar/Construir la base de datos con la información de interés.
  2. Realizar un análisis descriptivo de la información.
  3. Ajustar el modelo a través del ANOVA.
  4. Validar los supuestos del modelo.
  5. Realizar las pruebas post-hoc.

Necesitamos las siguientes librerías.

# Para gráficos con estilo
require(ggplot2)

# Para manejar bases de datos
require(dplyr)

# Para construir los qq-plots
require(car)

# Para las pruebas de normalidad
require(nortest)

# Para las pruebas post-hoc
require(agricolae)

El primer paso es cargar o construir la base de datos, esto puede hacerse en excel y luego importarlas a R o se puede construir la base de datos de la siguiente forma:

datos <- data.frame(respuesta=c(6,9,7,8,
                                7,10,11,8,
                                10,16,11,14,
                                10,13,11,9),
                    tratamientos=rep(LETTERS[1:4],each=4),
                    bloques=factor(rep(1:4,4)),
                    stringsAsFactors = T)

# verificar la tabla de datos
xtabs(respuesta~tratamientos+bloques,data=datos)

Los análisis descriptivos sirven para evaluar visualmente si existe un posible efecto del tratamiento en la variable respuesta, como no tenemos réplicas entonces la mejor opción es graficar los puntos y lineas que corresponden a los tratamientos con cada bloque.

ggplot(datos,aes(y=respuesta,x=bloques,colour=tratamientos))+

Luego se construye el modelo con el comando aov dando como argumentos la fórmula, después se ejecuta el comando anova para construir la tabla e interpretar.

# Anova
fit <- aov(respuesta~tratamientos+bloques,data=datos)
anova(fit)
